# P10-2 — Image async — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** URL `Image` with async load + cache + spinner + error fallback; `Spinner` widget.

Design ref: `docs/superpowers/specs/2026-07-03-phase10-2-image-design.md`.

**Tech Stack:** TypeScript, pnpm workspace, Vitest. Packages: host, platform-web, widgets, primitives.

---

### Task 1: `loadImage` host seam + platform-web loader
- Files: `packages/host/src/host.ts` (`loadImage?(url): Promise<ImageHandle>`), new `packages/platform-web/src/image-loader.ts` (`createImageLoader`, `clearImageCache`), `packages/platform-web/src/create-web-host.ts` (wire it), export from `packages/platform-web/src/index.ts`. Test `packages/platform-web/test/image-loader.test.ts`.
- `createImageLoader()`: closure `cache = new Map<string, Promise<ImageHandle>>()`; returns `(url) => { if (cache.has(url)) return cache.get(url)!; const p = new Promise<ImageHandle>((res, rej) => { const img = new Image(); img.onload = () => res(img as unknown as ImageHandle); img.onerror = () => rej(new Error('img load failed: '+url)); img.src = url; }); cache.set(url, p); return p; }`. `clearImageCache()` clears it (for tests). Guard `typeof Image !== 'undefined'`? In jsdom `Image` exists; in bare node it may not — the test should stub `globalThis.Image` or test cache identity with a fake.
- `create-web-host.ts`: `const loadImage = createImageLoader();` add `loadImage` to the returned Host.
- TDD: stub `globalThis.Image` with a class that stores `onload`/`src` and lets the test fire `onload`; assert `loadImage(url)` twice returns the SAME promise (cache); resolving fires with the handle. Use `clearImageCache()` in `beforeEach`.

### Task 2: `Spinner` widget
- Files: new `packages/widgets/src/spinner.ts` (`Spinner`, `SpinnerProps`), export from `packages/widgets/src/index.ts`. Test `packages/widgets/test/spinner.test.ts`.
- `Spinner({ size=24, color='#9ca3af', thickness=3 })`: `BoxNode(size)`; `[angle,setAngle]=createSignal(0)`; on creation start a loop `const loop = () => { setAngle(a => a + 0.15); handle = useHost().scheduler.requestFrame(loop); }; handle = useHost().scheduler.requestFrame(loop);` and `onCleanup(() => useHost().scheduler.cancelFrame(handle))`. `paintSelf(r)`: `const r0 = size/2 - thickness; const a = angle(); r.strokePath(createPath().arc(size/2, size/2, r0, a, a + Math.PI*1.5).build(), { color, width: thickness });`. Import `createPath` from `@cairn/host`, `useHost` from `@cairn/runtime`, `createSignal`/`onCleanup` from `@cairn/reactivity`.
- TDD (fake host with manual clock): building a Spinner requests a frame (scheduler.requestFrame called); `paintSelf` on a recording renderer calls `strokePath`; ticking advances the angle (a second paint uses a different arc start). Assert `strokePath` called + a frame requested.

### Task 3: `Image` async src
- Files: `packages/primitives/src/image.ts` (extend), test `packages/primitives/test/image-async.test.ts`.
- `ImageProps.src: ImageHandle | string`. Import `useHost` from `@cairn/runtime`, `createSignal`/`createEffect` from `@cairn/reactivity`, `scheduleFrame` from `@cairn/runtime`, `Spinner` — but widgets depends on primitives, so primitives CANNOT import from widgets (cycle). SOLUTION: put `Spinner` in `@cairn/primitives` instead of widgets (it's a low-level indicator, no state), OR have Image render a minimal inline spinner. DECISION: put `Spinner` in `@cairn/primitives` (Task 2 → create `packages/primitives/src/spinner.ts`, export from primitives; widgets can re-export it if desired). Update Task 2 location to primitives. Then Image imports `Spinner` from `./spinner`.
- When `typeof src === 'string'`: BoxNode gets `alignX/alignY:'center'`; signals `handle`/`status`('loading'); call `useHost().loadImage!(src).then(h => { setHandle(h); setStatus('loaded'); scheduleFrame(); }).catch(() => { setStatus('error'); scheduleFrame(); })`; a `createEffect` sets `instance.children = status()==='loading' ? [Spinner({ size: Math.min(width,height)/2 })] : []` and schedules a frame; `paintSelf`: `const h = handle(); if (h) drawImage fitted; else if (status()==='error') draw fallback (fill '#2a2a2e' rounded + a muted '×' or a light rect)`. When `src` is an `ImageHandle` (not string): existing synchronous path (draw directly, no children/signals).
- TDD (fake host with a controllable `loadImage`): `Image({ src:'u', width:100, height:100 })` → initially `instance.children.length===1` (spinner) and status loading; resolve the loadImage promise with a fake handle `{width:200,height:100}` → after a microtask flush, children clear and `paintSelf` on a recording renderer calls `drawImage`; reject → error (no drawImage, no throw). ImageHandle path: `Image({ src:{width:10,height:10} as any, ... })` → `paintSelf` draws immediately, no children.
