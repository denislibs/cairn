# NF2 — Native form controls (retrofit) — Implementation Plan

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development. TDD.

**Goal:** make `Checkbox`, `Switch`, `Radio`+`RadioGroup`, `Field` fully native (real roles/states/keyboard/AT) using the NF1 foundation.

Design: `docs/superpowers/specs/2026-07-04-nf-native-foundation-design.md`. Builds on NF1.

**Read first:** `packages/widgets/src/checkbox.ts`, `switch.ts`, `radio.ts`, `field.ts`; `packages/runtime/src/semantics.ts` (`SemanticsNode`); `packages/widgets/src/native/roving.ts` + `keys.ts` + `announce.ts`; `packages/widgets/src/control.ts`. `createEffect` from `@cairn/reactivity`, `scheduleFrame` from `@cairn/runtime`.

**Pattern for reactive semantics:** a component sets `instance.semantics = {...}` once, then keeps the changing fields (checked/disabled/label) in sync with a `createEffect(() => { instance.semantics!.checked = read(); })`. collectSemantics reads the live fields each frame (frames are already scheduled on state change).

---

### Task 1: `Checkbox` native
- On the returned (focusable) instance set `semantics = { role: 'checkbox', label: props.label, checked: read(), disabled, onActivate: toggle, onFocus:(kb)=>setFV(kb), onBlur:()=>setFV(false) }`. Keep an effect syncing `semantics.checked = read()` and `checked:'mixed'` when `indeterminate`. Focus-visible ring on keyboard focus (use `createControl` focusVisible or a local signal like Button). Space toggles (bridge handles Enter/Space→onActivate for role checkbox). Keep pointer/click working.
- Tests: instance.semantics role checkbox; checked reflects state and updates after toggle (read the field after calling onActivate); indeterminate→'mixed'; disabled carried; onActivate toggles + blocked when disabled.
- Commit: `feat(widgets): Checkbox native semantics (role/aria-checked/space)`.

### Task 2: `Switch` native
- `semantics = { role:'switch', label, checked: read(), disabled, onActivate: toggle, onFocus, onBlur }` + effect syncing checked. Focus-visible ring.
- Tests: role switch; aria checked updates; disabled; onActivate.
- Commit: `feat(widgets): Switch native semantics (role switch/aria-checked)`.

### Task 3: `Radio` + `RadioGroup` native (roving)
- `RadioGroup`: set `semantics = { role:'radiogroup', label?: props.label, disabled }` on its container. Provide (in the existing context) an active-index + `handleArrow(key)` using `createRoving({ count: () => registeredCount, orientation:'both', loop:true, initial: checkedIndex })`. Selecting via arrow ALSO sets the value (radio semantics: arrows move+select).
- `Radio`: `semantics = { role:'radio', label, checked: ()=>groupValue===value (synced via effect), disabled, focusable: isActive, autoFocus: isActiveAndKeyboardMoved, onActivate: ()=>select(value), onKeyDown: (key)=> group.handleArrow(key) /* returns handled */ }`. Roving: only the active radio has `focusable:true` (tabindex 0), others -1; when arrows change active, set that radio's `autoFocus:true` (transient) so the bridge moves focus. `onActivate` (Enter/Space/click) selects. Arrow via onKeyDown → group moves active + selects + sets autoFocus.
- Keep a per-radio effect syncing `semantics.checked` and `semantics.focusable`/`autoFocus`.
- Tests: RadioGroup instance role radiogroup; each Radio role radio + checked reflects selection; selecting sets value; arrow onKeyDown moves selection to next/prev value (and returns handled); only one radio focusable at a time (focusable flags).
- Commit: `feat(widgets): Radio/RadioGroup native (radiogroup+radio, aria-checked, roving)`.

### Task 4: `Field` label association + error announce
- `Field.Label`: register the label text into the field context (add `label` to the context value). Controls that read `useFieldOptional()` (e.g. `Input`) should default their `semantics.label`/description from the field when they have no explicit label — do the wiring in Field's context (add `labelText` + `describedBy`/error text to context) and document that Input picks it up in NF3.
- `Field.Error`: when it becomes visible (invalid()), announce it politely via `useAnnounce()` (so screen readers hear the validation error). Guard so it announces on change, not every frame.
- Tests: Field context exposes label text; Field.Error calls announce when invalid flips true (mock host.a11y.announce).
- Commit: `feat(widgets): Field label association + error announcements`.

### Task 5: demo + browser a11y verify
- Extend `examples/forms` (or `examples/a11y`) with the retrofitted controls; ensure the canvas parent is positioned (a11y overlay).
- Playwright: `browser_snapshot` → assert a11y tree shows `checkbox "..." [checked]`, `switch`, `radiogroup` with `radio [checked]`; keyboard: Space toggles the checkbox (dispatch or key), arrows move the radio selection; screenshot. Full `pnpm test` + `pnpm typecheck` green.
- Commit: `docs(a11y): NF2 native form-controls demo + verification`.
