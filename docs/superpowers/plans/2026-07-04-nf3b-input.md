# NF3b — Native text Input (textbox a11y) — Implementation Plan

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development. TDD. IMPORTANT: do NOT break existing Input behavior/tests — the new path is guarded by `host.a11y` being present; the existing text-seam path stays for the no-a11y case.

**Goal:** the text `Input` appears in the a11y tree as a real `role=textbox` with name/value/state, and is edited via a real per-field `<input>` (native focus/IME/selection) when the a11y bridge is present. Falls back to the existing platform text-seam when there is no bridge.

Design: `docs/superpowers/specs/2026-07-04-nf-native-foundation-design.md`. Builds on NF1-3.

**Read first:** `packages/primitives/src/input.ts` (the field: text()/caret()/focused() signals, controlled/uncontrolled, seam via `useHost().textInput.start`, canvas paint of text+caret+placeholder). `packages/platform-web/src/web-accessibility.ts` (bridge; currently SKIPS role 'textbox'). `packages/host/src/accessibility.ts`, `packages/runtime/src/semantics.ts`. `packages/platform-web/src/web-text-input.ts` (seam, for reference).

---

### Task 1: semantics fields for textbox (host + runtime)
- `SemanticsNodeData` + `SemanticsNode`: add `onInput?: (value: string) => void`, `placeholder?: string`, `multiline?: boolean`. (`value`, `readonly`, `disabled`, `label` already exist.) Carry through in `collectSemantics`.
- Test: collectSemantics carries onInput/placeholder/multiline.
- Commit: `feat(host/runtime): semantics textbox fields (onInput/placeholder/multiline)`.

### Task 2: bridge — real editable textbox element
- `web-accessibility.ts`: STOP skipping role 'textbox'. For it, `createElement`: `<textarea>` if `multiline` else `<input type=text>`. In `updateAttributes`: set `aria-label`/placeholder/readOnly/disabled/tabindex/position as usual; set the element's `.value = node.value` **only when the element is NOT focused** (don't clobber the caret while typing). Add an `input` listener → `node.onInput?.(el.value)` (via the getState latest-node pattern). Keep focus/blur/keydown (onKeyDown forwarding covers Enter/Escape). The element is the real editor + a11y node.
- Tests (jsdom): sync a textbox node → an `<input>` with aria-label/placeholder/value/tabindex; dispatching 'input' calls onInput with the value; `multiline:true` → `<textarea>`; re-sync with a new value updates `.value` when not focused; `readonly`/`disabled` reflected.
- Commit: `feat(platform-web): bridge renders a real editable textbox (input/textarea) for role textbox`.

### Task 3: `Input` declares textbox semantics + a11y editing path (primitives)
- In `packages/primitives/src/input.ts`: read `const host = useHost()`. Set on the returned instance:
  `instance.semantics = { role:'textbox', label: props.label ?? props.placeholder-as-fallback?, value: text(), placeholder, disabled, readonly, onInput: (v)=>{ setText(v); setCaret(v.length); props.onInput?.(v); }, onKeyDown: (key)=> { if(key==='Enter'){ props.onSubmit?.(untrack(text)); return true } return false }, onFocus:(kb)=>{...}, onBlur:()=>{...} }`. Keep `value`/`disabled`/`readonly` synced via `createEffect`.
  - `label`: `props.label` if the Input has one (add an optional `label?: string`/`ariaLabel?: string` prop), else the placeholder as a weak fallback. (widgets `Input` frame / `Field` can pass a label later.)
  - **Editing path**: if `host.a11y` is present → do NOT open the seam session on focus (the bridge's real `<input>` is the editor; typing flows via `onInput`). If `host.a11y` is absent → keep the EXISTING seam path exactly as-is. Guard the focus handler accordingly.
  - Canvas still paints text()/caret()/placeholder as today (the real input is transparent).
- Tests: with a fake host WITHOUT a11y → existing seam behavior unchanged (all current input tests pass). With a fake host WITH a11y (a11y.sync stub) → instance.semantics role textbox, value reflects text(), onInput updates text()+fires props.onInput, onKeyDown Enter fires onSubmit; seam NOT started on focus.
- Commit: `feat(primitives): Input native textbox semantics + a11y editing path (seam fallback kept)`.

### Task 4: verify (controller does browser)
- Full `pnpm test` + `pnpm typecheck` green. Browser (`examples/forms`): a11y tree shows `textbox "Name"`/`"Email"`; typing into the real input updates the canvas text; the invalid Email field still shows its error. Confirm existing seam editing still works in an example without a11y is N/A (all our examples use the web host which now has a11y) — so just confirm editing works via the a11y input path in the browser.
- Commit: `docs(a11y): NF3b Input textbox verification`.
