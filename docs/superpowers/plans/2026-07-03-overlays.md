### Task 1: OverlayRegistry + overlayContext + mount overlay layer
- Files: new `packages/runtime/src/overlays.ts` (`OverlayRegistry`, `createOverlayRegistry`, `overlayContext`, `useOverlays`), export from `packages/runtime/src/index.ts`; modify `packages/runtime/src/mount.ts`. Tests `packages/runtime/test/overlays.test.ts`, extend/create `packages/runtime/test/mount-overlays.test.ts`.
- `createOverlayRegistry()`: internal `createSignal<{id,inst}[]>([])`; `add(inst)` → push `{id:next++, inst}`, return id; `remove(id)` → filter; `list()` → `sig().map(e=>e.inst)` (reactive read); also `appRoot(): Instance | null` + `setAppRoot(inst)` (mount sets it). `overlayContext = createContext<OverlayRegistry | null>(null)`; `useOverlays()` = `useContext(overlayContext)` (throw if null with a helpful message).
- `mount.ts`:
  - `const overlays = createOverlayRegistry();`
  - build app root under BOTH contexts: `root = runWithContext(hostContext, host, () => runWithContext(overlayContext, overlays, () => component()))` (verify runWithContext nesting works; else use a combined provider). Then `overlays.setAppRoot(root)`.
  - `layered()` returns `{ layout: { offsetX:0, offsetY:0, size:{ w: host.metrics.width, h: host.metrics.height } } as any, children: [root, ...overlays.list()], handlers: {}, paintSelf(){} } as Instance`.
  - `renderFrame`: set ctx.viewport; layout root tight surface (as today); for each `o of overlays.list()` → `o.layout.layout({minW:0,maxW:w,minH:0,maxH:h}, ctx)`; `beginFrame; clear; paint(root); for (o of overlays.list()) paint(o, renderer); endFrame`.
  - dispatcher + focus `getRoot` → `layered`.
- TDD: registry add/list/remove; `useOverlays` returns provided registry under `runWithContext`. mount-overlays: mount an app with a Portal-like registration → after render, a hit at the overlay's rect returns the overlay; paint visits app root then overlay (use a recording renderer + spy paintSelf order). (Can test the `layered()`/registry behavior without a full DOM host via the runtime fake host.)

### Task 2: Portal primitive + positioning engine
- Files: new `packages/primitives/src/portal.ts` (`Portal`), new `packages/primitives/src/placement.ts` (`computePlacement`, `getAbsRect`, types), export from `packages/primitives/src/index.ts`. Tests `packages/primitives/test/portal.test.ts`, `packages/primitives/test/placement.test.ts`.
- `Portal({ children })`: `const reg = useOverlays(); const id = reg.add(children); onCleanup(() => reg.remove(id));` return placeholder `{ layout: new BoxNode({ width:0, height:0 }), children: [], paintSelf(){} }`.
- `computePlacement(anchor, content, viewport, {side='bottom', align='center', offset=8, flip=true})`: compute primary x/y for the side; if flip and overflow on that side, use opposite; align along the cross axis (start/center/end); clamp x∈[0,vw-content.w], y∈[0,vh-content.h]; return {x,y,side}.
- `getAbsRect(target, root)`: DFS accumulating offsets; on match return `{x,y,width:size.w,height:size.h}`; else null.
- TDD: placement bottom (`y = anchor.y+anchor.h+offset`, centered x); flip top when no room below; clamp; getAbsRect nested (root>a>b) sums offsets; not-found → null. Portal: registering adds to a fake registry provided via context; placeholder zero-size; dispose removes.

### Task 3: Modal (@cairn/widgets)
- Files: new `packages/widgets/src/modal.ts` (`Modal`), export from `packages/widgets/src/index.ts`. Test `packages/widgets/test/modal.test.ts`.
- `Modal({ open, onClose, children })`: `open` bool|accessor. Use `Show`(from runtime) on `open` to mount a `Portal` whose content = full-surface `Box` (`style:{ width:'100%', height:'100%', backgroundColor:'rgba(0,0,0,0.5)', alignX:'center', alignY:'center' }`, `onClick: onClose`, focusable, `onKeyDown: e => e.key==='Escape' && onClose()`) containing a content wrapper Box with `onClick` that stops propagation (call `e.stopPropagation?.()` if available, else guard: backdrop onClick checks the event target — simplest: content Box has its own onClick `() => {}` and rely on hit-test topmost = content, so backdrop onClick only fires when clicking the dim area; verify with the events model — the click target is the topmost node, so clicking content dispatches to content first and bubbles to backdrop → need stopPropagation). Check `CairnPointerEvent` for `stopPropagation`; if present use it, else document the limitation and use a flag.
- TDD: `open=false` → no overlay registered (Show renders nothing); `open=true` → Portal registered (1 overlay); backdrop onClick → onClose called; Escape → onClose. (Use a fake overlay registry via context in the test.)

### Task 4: Tooltip + Popover (@cairn/widgets)
- Files: new `packages/widgets/src/tooltip.ts`, `packages/widgets/src/popover.ts`, export from index. Tests `packages/widgets/test/tooltip.test.ts`, `popover.test.ts`.
- Both wrap a `trigger`/`children` Instance inline and hold a visibility signal. On show, render a `Portal` with a full-surface `Stack` containing the bubble/content `Box` positioned via `left/top` from `computePlacement(getAbsRect(trigger, useOverlays().appRoot()!), size, viewport, {...})`. Viewport from `useViewport()` (S5) or `useHost().metrics`.
- Tooltip: trigger `onPointerEnter` → show, `onPointerLeave` → hide.
- Popover: trigger `onClick` → toggle; the Portal content includes a full-surface transparent catcher Box (`onClick` → close) behind the content; Escape → close.
- Since real geometry needs a layout pass, tests focus on the CONTRACT: hover/click toggles the overlay-registered count; outside-catcher click closes Popover; Escape closes. (Positioning correctness is covered by `computePlacement` unit tests + the browser check.)
- TDD: Tooltip hover shows/hides overlay; Popover click toggles; catcher-click closes.

## Self-review
- Coverage: registry+mount (T1), Portal+placement (T2), Modal (T3), Tooltip+Popover (T4), doc+example (T5).
- Risk: mount `getRoot` now returns a synthetic layered root — existing hit/focus tests use `mount` with no overlays; `layered()` with empty overlays = `{children:[appRoot]}`, so a hit descends into appRoot as child[0] — behavior identical (one extra wrapper level with offset 0). Verify existing events/focus-through-mount tests stay green.
- Risk: `stopPropagation` on pointer events — check `packages/events/src/event.ts`/`dispatch.ts`; the click dispatch bubbles target→root, so Modal content-vs-backdrop needs it. If absent, add `stopPropagation()` to the dispatched event (small events change) or use hit-target check.
- Backward-compat: no Portal used → registry empty → mount paints app root then nothing; identical output.
