### Task 3: `Radio` + `RadioGroup` (compound)
- New `packages/widgets/src/radio.ts` (both). Export from index. Test `packages/widgets/test/radio.test.ts`.
- `createCompoundContext('RadioGroup')` value type `{ value: Accessor<any>; setValue: (v:any)=>void; disabled: boolean }`.
- `RadioGroup({ value?, defaultValue?, onChange?, disabled?, children })`: controlled/uncontrolled value signal; provides context via `Provider`; renders a Column (theme gap). 
- `Radio({ value, disabled?, label?, style?, children? })`: `const g = useRadioGroup()`; `checked = () => g.value() === props.value`; select on click/Space (guarded by group.disabled||props.disabled) → `g.setValue(props.value)`. Default look: outer ring (border) + inner dot (bg primary) when checked; focus ring. Arrow-key roving: `onKeyDown` Arrow Down/Right → select next sibling value, Up/Left → prev (the group can expose an ordered list of registered values; simplest: RadioGroup collects child values via context registration, Radio registers its value on construction). Keep roving simple and tested.
- TDD: selecting sets group value + onChange; controlled group (value fixed) still fires onChange, internal unused; only one checked at a time (two radios, select one → other unchecked); disabled group blocks; `useRadioGroup()` throws outside a group; arrow key moves selection.
- Commit: `feat(widgets): Radio + RadioGroup (compound, roving keyboard)`.

### Task 5: `Input` (styled text field)
- New `packages/widgets/src/input.ts` (name `Input`; note it shadows primitives `Input` — widgets consumers import from `@cairn/widgets`). Export from index. Test `packages/widgets/test/input.test.ts`.
- Wrap `Input as TextField` from `@cairn/primitives`. Structure: outer frame `Box` (default: bg surface, border borderStrong, radii.md, padding control.padX/height, `focus`-ring when the inner field is focused, `disabled` dim, `invalid`→danger border) containing the primitives Input styled transparent (`{ backgroundColor:'transparent', width:'100%', color: text, font }`). 
- Lift focus: track a `focused` signal; pass `onFocus`/`onBlur` to the inner primitives Input to set it; the frame Box uses a **function-form style** reading `focused()` (and `invalid`) to apply the focus-ring/border reactively (the frame Box is NOT focusable — only the inner field is).
- Props: `value?`, `defaultValue?`, `onInput?`, `onChange?` (alias of onInput firing on each change), `onSubmit?`, `placeholder?`, `disabled?`, `invalid?`, `size?`, `style?` (frame), `inputStyle?` (inner). Read `useFieldOptional()` → if present, default `invalid`/`disabled` from the field.
- Controlled/uncontrolled: pass `value` through to the primitives Input (it already supports controlled `value` + `onInput`); `defaultValue` seeds an internal uncontrolled signal passed as `value` only if you manage it — simplest: if `value===undefined` use uncontrolled (let primitives Input manage), forward onInput.
- disabled: when disabled, the frame sets `pointerEvents:'none'`-ish (don't focus) — at minimum dim + skip focusing; document if full disable of the seam is deferred.
- TDD (use the primitives Input test harness for hostContext + a fake textInput seam): onInput fires when the inner client onChange runs; controlled value syncs; focus lifts (after inner onFocus, the frame style resolves the focus-ring — assert by resolving the frame's function-style with focused=true, or assert the focused signal flips); reads Field invalid (build inside a Field with invalid=true → frame invalid style).
- Commit: `feat(widgets): Input — themed text field frame over the platform text seam`.
