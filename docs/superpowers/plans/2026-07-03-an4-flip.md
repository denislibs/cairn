# AN4 — FLIP list animations — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** `For` with `flip` animates keyed item moves via a post-layout hook + invert/play transforms.

**Tech Stack:** TypeScript, pnpm workspace, Vitest. Packages: runtime.

### Task 1: post-layout hook (@cairn/runtime)
- Files: `packages/runtime/src/scheduler.ts` (add `onNextLayout` + `flushAfterLayout`), `packages/runtime/src/mount.ts` (call `flushAfterLayout()` after layout, before paint), export `onNextLayout` from `packages/runtime/src/index.ts`. Test `packages/runtime/test/after-layout.test.ts`.
- scheduler.ts:
```ts
let afterLayout: Array<() => void> = [];
export function onNextLayout(cb: () => void): void { afterLayout.push(cb); scheduleFrame(); }
export function flushAfterLayout(): void { const cbs = afterLayout; afterLayout = []; for (const cb of cbs) cb(); }
```
- mount.ts `renderFrame`: after `root.layout.layout(...)` and the overlay layout loop, and BEFORE `host.renderer.beginFrame()`/`paint`, call `flushAfterLayout()`.
- TDD: `onNextLayout(cb)` → after `flushAfterLayout()` cb ran once; a second `flushAfterLayout()` does not re-run it; two callbacks both run.

### Task 2: `For` FLIP
- Files: `packages/runtime/src/for.ts` (add `flip` handling), test `packages/runtime/test/for-flip.test.ts`.
- `ForProps<T>` gains `flip?: boolean | { duration?: number; easing?: EasingName | EasingFn; spring?: { stiffness?: number; damping?: number; mass?: number } }`.
- In the reconcile effect, when `flip` is set and this is not the initial run (there were prior entries): BEFORE `apply(ordered)`, snapshot surviving entries' offsets: `const prev = new Map<key, {x,y}>()` from `entry.instance.layout.offsetX/offsetY` for keys present in both old `entries` and `next`. After `apply(ordered)`, `onNextLayout(() => { for (const [k, p] of prev) { const inst = next.get(k)!.instance; const dx = p.x - inst.layout.offsetX, dy = p.y - inst.layout.offsetY; if (dx === 0 && dy === 0) continue; cancelFlip(k); inst.transform = { translateX: dx, translateY: dy }; const opts = normalizeFlip(props.flip); const from = { translateX: dx, translateY: dy }; flips.set(k, animate({ from: 0, to: 1, duration: opts.duration, easing: opts.easing, onUpdate: (t) => { inst.transform = { translateX: from.translateX * (1 - t), translateY: from.translateY * (1 - t) }; scheduleFrame(); }, onDone: () => { inst.transform = undefined; flips.delete(k); } })); } })`.
  - Keep a `flips: Map<key, () => void>` of in-flight cancels; `cancelFlip(k)` calls+deletes. On dispose/cleanup, cancel all.
  - Use `animate` from `./animate` (progress 0→1, invert*（1-t)). (Spring variant optional: if `opts.spring`, use `animateSpring` similarly.)
  - `normalizeFlip(flip)`: `flip === true` → `{ duration: 250, easing: 'ease-out' }`; object → merge defaults.
- TDD (fake manual-clock host): set up a For with 3 keyed items; manually assign `entry.instance.layout.offsetY` to simulate an initial layout (e.g. 0, 50, 100); trigger a reorder (change `each()`); in the reconcile the snapshot captures old offsets; then simulate new layout by setting new offsets and calling `flushAfterLayout()` — assert a moved item got `transform` set to the inverted delta; drive the clock → transform settles to `undefined`/~0. (You control offsets directly for determinism; the test verifies invert+play, not real Flex geometry.)
